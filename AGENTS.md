# AGENTS.md - Agentic Coding Guidelines

本文件为在此代码库工作的智能体提供编码指南。

## 项目概述

这是一个基于大模型的个人助手 CLI 工具，采用 **ReAct (Reasoning + Acting)** 架构：

```
用户输入 → 意图识别 → 生成行动计划 → 用户确认 → 执行循环 → 返回结果
```

### 详细流程说明

#### 1. 意图识别
- 判断是闲聊还是需要执行工具
- 闲聊：直接调用 LLM3 生成回复
- 工具任务：进入 ReAct 模式

#### 2. 生成行动计划
模型分析用户需求，生成行动计划：
```
行动计划:
  1. [ ] 步骤1描述
  2. [ ] 步骤2描述
  3. [ ] 步骤3描述

用户确认是否执行 (y/n):
```

#### 3. 执行循环（ReAct Loop）

**成功时**：
```
执行第N步: [工具名称]
结果: [执行结果]
→ 检查是否还有未执行步骤
→ 有：继续循环
→ 无：进入返回结果
```

**失败时**：
```
执行第N步: [工具名称]
错误: [错误信息]

如何处理?
  [1] 重试 - 请输入修改方案或参数：
  [2] 跳过 - 忽略此步骤，继续下一步
  [3] 终止 - 停止整个计划

请选择:
```

#### 4. 返回结果
- 汇总所有成功步骤的结果
- 列出失败的步骤及错误
- 模型生成自然语言总结

### 循环终止条件
- 所有步骤执行完成
- 用户选择终止
- 达到最大循环次数（10次）

## 代码风格指南

### 导入顺序
- 标准库 → 第三方库 → 本地模块
- 使用 `src` 的绝对导入
```python
import os
import sys

import requests
import yaml

from src.config.loader import get_config
from src.model.factory import LLMFactory
```

### 格式化
- 使用 4 空格缩进（不使用 Tab）
- 最大行长度：120 字符
- 逻辑区块之间使用空行

### 类型提示
- 所有函数参数和返回值使用类型提示
- 使用 `Optional[X]` 保持兼容性
- 使用 `typing` 模块的类型
```python
def process(self, user_input: str) -> str:
    tools: Dict[str, Any] = {}
    session_history: Optional[str] = None
```

### 命名规范
- **类**: PascalCase（如 `IntentService`、`LLMFactory`）
- **函数/变量**: snake_case（如 `get_config`、`user_input`）
- **常量**: UPPER_SNAKE_CASE
- **私有方法**: 前缀下划线（如 `_build_tool_list`）

### 文档字符串
- 使用中文（符合项目语言）
- 复杂方法包含 Args 和 Returns 说明
```python
def recognize(self, user_input: str, session_history: Optional[str] = None) -> Tuple[str, str]:
    """
    识别用户意图，返回意图类型和工具名称

    Args:
        user_input: 用户输入的自然语言
        session_history: 会话历史（可选）

    Returns:
        (intent_type, tool_name): 意图类型和工具名称
    """
```

### 错误处理
- 优先使用具体异常类型
- 记录错误上下文后重新抛出
- 工具失败时返回用户友好的错误信息

### 日志
- 使用 `src.logging.setup` 中的日志系统
- 模块级别获取 logger：`logger = get_logger("ClassName")`
- 使用适当级别：DEBUG、INFO、WARNING、ERROR

### 配置
- 配置存储在 `config.yaml`
- 使用 `Config` 类访问配置
- 密钥使用环境变量

### 工具开发
1. 在 `src/tools/` 目录创建 Tool 类
2. 继承 `Tool` 基类
3. 实现必要属性：`name`、`description`、`params_schema`、`execute`
4. 可选实现 `prompt_hint` 用于参数生成
5. 在 `src/cli.py` 中注册工具

### 服务开发
- 服务处理业务逻辑和 LLM 交互
- 每个服务类在 `__init__` 中接受相关依赖
- 使用依赖注入提高可测试性
